# 变更检查清单

提交一个后端改动前，至少确认：

- 改动落点是否在正确分层
- 是否影响了跨模块调用链
- 是否需要同步更新模块文档
- 是否需要补测试或架构约束
- 是否影响初始化脚本、配置或权限
- 是否引入了无用/冗余代码，能否归类为 `P0 delete`、`P1 verify then delete`、`P2 consolidate`
  或 `Keep`
- 如果保留兼容 wrapper、mock 脚本或跨模块 model 读取，是否有明确的运行期调用方或架构测试例外

如果改动涉及以下内容，要额外检查：

- workflow：配置与代码是否一致
- test_specs：状态投影是否仍正确
- execution：当前态与事件归档是否一致
- auth：权限依赖是否仍准确
