# 测试需求与测试用例后端设计说明（精简版）

更新时间：2026-02-28

## 1. 文档定位

本文档只描述后端实现关注点，不再重复产品级字段定义和 BOM 业务背景。

权威文档入口：

- 字段定义：`../../docs/测试需求字段定义.md`、`../../docs/测试用例字段定义.md`
- 总体业务背景：`../../docs/测试设计与BOM关联方案.md`
- 后端接口返回规范：`../../docs/后端接口说明.md`

## 2. 后端落地范围

后端模块：`app/modules/test_specs/`

- 需求路由：`/api/v1/requirements`
- 用例路由：`/api/v1/test-cases`
- 核心关系：`TestCase.ref_req_id -> Requirement.req_id`

## 3. 关系约束（当前实现）

1. 创建/更新用例时，必须校验 `ref_req_id` 对应需求存在。
2. 支持按 `ref_req_id` 查询用例列表。
3. 需求删除策略应避免悬挂用例（由服务层策略控制）。

更多细节见：`requirement_testcase_relation_design.md`

## 4. 典型接口映射

### 需求

- `POST /api/v1/requirements`
- `GET /api/v1/requirements`
- `GET /api/v1/requirements/{req_id}`
- `PUT /api/v1/requirements/{req_id}`
- `DELETE /api/v1/requirements/{req_id}`

### 用例

- `POST /api/v1/test-cases`
- `GET /api/v1/test-cases`
- `GET /api/v1/test-cases/{case_id}`
- `PUT /api/v1/test-cases/{case_id}`
- `DELETE /api/v1/test-cases/{case_id}`

## 5. 权限要求

- 需求接口：`requirements:read` / `requirements:write`
- 用例接口：`test_cases:read` / `test_cases:write`

权限矩阵见：`authorization_design.md`。

## 6. 演进建议

1. 若出现“一个用例覆盖多个需求”的业务场景，引入关联表。
2. 增加关系变更审计字段，便于回溯。
3. 对高风险操作补充约束测试（删除需求、跨需求迁移用例）。
