# 开发与架构约定

## 分层约定

- API：参数接收、依赖注入、响应格式
- Application / Service：业务用例与资源逻辑
- Repository：数据访问与文档模型
- Domain：规则、策略、异常

## 代码风格

- Python 最大行长 `110`
- 复杂度上限 `12`
- 优先绝对导入
- 使用 async/await 处理 I/O

## 设计约定

- API 不直接操作数据库
- `status` 投影字段不能直接更新
- 删除优先考虑 soft delete 与跨模块一致性
- 新增兼容 facade 前先证明没有更小的明确接口可用

## 工程约定

- 重要架构约束应有测试保护
- 模块 README 只能当摘要，完整实现细节要沉淀到这里的文档站
