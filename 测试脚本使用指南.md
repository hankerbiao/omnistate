## ❌ 常见错误及解决方案

### 错误 1: 导入失败

```
✗ 导入失败: No module named 'app'
```

**解决方案:**
```bash
# 确保在项目根目录运行
cd /Users/libiao/Desktop/github/dmlv4

# 检查路径
ls backend/app/shared/kafka/kafka_message_manager.py
```

### 错误 2: Kafka 连接失败

```
Kafka 生产者: ✗ 失败
```

**解决方案:**
```bash
# 检查 Kafka 服务状态
telnet 10.17.154.252 9092

# 检查 Kafka 日志
docker logs kafka  # 如果使用 Docker

# 验证网络连通性
ping 10.17.154.252
```

### 错误 3: 认证失败（仅完整测试）

```
✗ 无法获取认证令牌
```

**解决方案:**
```bash
# 方法 1: 创建测试用户
cd backend
python scripts/create_user.py

# 方法 2: 使用现有用户（编辑脚本中的 test_user 字典）
# 或设置环境变量
export TEST_TOKEN=your_jwt_token_here
```

### 错误 4: 测试用例不存在（仅完整测试）

```
KeyError: "case not found: ['TC001']"
```

**解决方案:**
```bash
# 查询数据库中的用例 ID
# 方法 1: 通过 API
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8000/api/v1/test-cases | jq '.data[].case_id'

# 方法 2: 直接修改测试脚本
# 编辑 test_kafka_integration.py 中的 TEST_CASES 变量
TEST_CASES = [
    {"case_id": "实际存在的用例ID1"},
    {"case_id": "实际存在的用例ID2"},
]
```

### 错误 5: 权限不足（仅完整测试）

```
HTTPException: 403 Forbidden - execution_tasks:write permission required
```

**解决方案:**
```bash
# 为用户分配权限
cd backend
python scripts/init_rbac.py  # 重新初始化 RBAC
```
