# 前后端字段统一说明（登录 / 创建测试需求 / 创建测试用例）

## 目标
- 登录、创建测试需求、创建测试用例三条链路，前后端字段名必须一致。
- 前端不做字段重命名或映射转换（例如 `required_env.tooling -> tooling_req` 这种转换已移除）。
- 创建接口仅提交后端 `Create*Request` 定义字段，不夹带服务端生成字段。

## 1. 登录字段统一

### 请求字段（`POST /api/v1/auth/login`）
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `user_id` | string | 是 | 用户登录 ID |
| `password` | string | 是 | 登录密码 |

### 响应字段（`APIResponse<LoginResponse>.data`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `access_token` | string | JWT |
| `token_type` | string | Bearer |
| `user` | object | 当前用户信息 |
| `user.id` | string | 数据库主键 |
| `user.user_id` | string | 用户业务 ID |
| `user.username` | string | 用户名 |
| `user.email` | string \| null | 邮箱 |
| `user.role_ids` | string[] | 角色列表 |
| `user.status` | string | 用户状态 |
| `user.created_at` | string(datetime) | 创建时间 |
| `user.updated_at` | string(datetime) | 更新时间 |

| 字段 | 类型 | 必填 | 说明 |

| `target_components` | string[] | 否 | 目标部件 |
| `firmware_version` | string \| null | 否 | 固件版本 |
| `priority` | string | 否 | 优先级 |
| `key_parameters` | {key:string,value:string}[] | 否 | 关键参数 |
| `risk_points` | string \| null | 否 | 风险点 |
| `tpm_owner_id` | string | 是 | TPM 负责人 |

| `auto_dev_id` | string \| null | 否 | 自动化开发人 |

### 创建时禁止提交字段
- `status`
- `created_at`
- `updated_at`

以上字段由后端生成或维护。

## 4. 已落地统一规则
- 前端 `CreateRequirementPayload` 与 `CreateTestCasePayload` 只保留后端创建字段。
- 登录本地降级模式也按 `user_id` 校验，不再按 `username` 兜底。
- 用例工具字段统一为顶层 `tooling_req`，不再使用 `required_env.tooling`。
- 用例状态、需求状态统一为后端工作流状态码（如 `DRAFT`、`PENDING_REVIEW`）。
