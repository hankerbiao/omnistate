# API 接口概览

## 基础信息

- API 基础地址：`http://localhost:8000/api/v1`
- 认证方式：JWT Bearer Token
- 数据格式：`application/json`

## 当前统一响应格式

当前后端统一 Envelope 以代码实现为准：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

说明：

- `code=0` 表示接口业务成功。
- HTTP 状态码仍用于表达 `401`、`403`、`404`、`422`、`500` 等错误类别。
- 当前统一响应模型不包含 `timestamp` 字段。

## 接口分组

### 核心业务模块

| 模块 | 前缀 | 说明 |
|------|------|------|
| 工作流管理 | `/work-items` | 业务事项状态流转与日志查询 |
| 测试需求 | `/requirements` | 测试需求 CRUD |
| 测试用例 | `/test-cases` | 测试用例 CRUD |
| 测试执行 | `/execution` | 执行代理、任务下发、状态回传 |

### 支撑模块

| 模块 | 前缀 | 说明 |
|------|------|------|
| 资产管理 | `/assets` | 部件库、DUT、计划关联部件 |
| 认证授权 | `/auth` | 登录、用户、角色、权限、导航 |
| 健康检查 | `/health` | 存活、就绪、基础健康检查 |

## 常见错误

| HTTP 状态码 | 含义 |
|-------------|------|
| `400` | 请求参数或业务前置条件不满足 |
| `401` | 未认证或凭证无效 |
| `403` | 已认证但权限不足 |
| `404` | 目标资源不存在 |
| `409` | 资源冲突 |
| `422` | 请求体验证失败 |
| `500` | 服务内部错误 |

## 维护约定
