# 登录接口字段说明文档

## 概述

本文档详细说明前端登录页面连接真实后端 API 所需的字段、数据流程及实现细节。

## 前端登录表单字段

前端登录组件 `Login.tsx` 中定义的表单字段：

```typescript
interface LoginForm {
  username: string;    // 用户名
  password: string;    // 登录密码
  rememberMe: boolean; // 是否记住登录状态
}
```

### 字段说明

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `username` | string | 是 | 用户名，用于显示和用户输入 |
| `password` | string | 是 | 登录密码 |
| `rememberMe` | boolean | 否 | 是否在客户端记住登录状态（前端本地存储） |

## 后端登录接口

### 请求信息

- **接口路径**: `POST /api/v1/auth/login`
- **Content-Type**: `application/json`

### 请求体字段

后端登录接口使用的 `LoginRequest` 模型：

```python
class LoginRequest(BaseModel):
    user_id: str      # 用户唯一标识（注意：不是 username）
    password: str     # 登录密码
```

**重要差异说明**：
- 前端使用 `username` 进行登录
- 后端接口期望接收 `user_id`（系统内部用户标识）
- `user_id` 与 `username` 是两个不同的字段

### 请求示例

```json
{
  "user_id": "admin",
  "password": "123456"
}
```

## 字段映射关系

### 前端 → 后端转换

前端登录时需要将 `username` 映射为 `user_id` 进行后端调用：

| 前端字段 | 后端字段 | 转换说明 |
|----------|----------|----------|
| `username` | `user_id` | 直接传递用户名作为 user_id（当前系统设计） |
| `password` | `password` | 保持一致 |

### 示例代码（前端）

```typescript
// 登录调用转换
const handleLogin = async () => {
  try {
    // 前端表单
    const loginForm = {
      username: 'admin',
      password: '123456',
      rememberMe: false
    };

    // 转换为后端接口期望的格式
    const requestBody = {
      user_id: loginForm.username,  // username 作为 user_id
      password: loginForm.password
    };

    // 调用后端 API
    const response = await fetch('/api/v1/auth/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(requestBody)
    });

    const result = await response.json();
    // 处理响应...
  } catch (error) {
    // 错误处理...
  }
};
```

## 响应数据结构

### 成功响应

后端登录成功返回 `LoginResponse`：

```python
class LoginResponse(BaseModel):
    access_token: str      # JWT 访问令牌
    token_type: str = "Bearer"  # 令牌类型
    user: UserResponse     # 用户信息
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "Bearer",
    "user": {
      "id": "64a7b8c9d1e2f3a4b5c6d7e8",
      "user_id": "admin",
      "username": "管理员",
      "email": "admin@example.com",
      "role_ids": ["admin_role"],
      "status": "ACTIVE",
      "created_at": "2024-01-01T00:00:00Z",
      "updated_at": "2024-01-01T00:00:00Z"
    }
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `access_token` | string | JWT 访问令牌，用于后续 API 鉴权 |
| `token_type` | string | 令牌类型，固定值 `"Bearer"` |
| `user` | object | 用户详细信息对象 |

#### user 对象字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 数据库主键 ID |
| `user_id` | string | 用户唯一标识（登录名） |
| `username` | string | 用户显示名称 |
| `email` | string | 用户邮箱（可选） |
| `role_ids` | array | 用户角色 ID 列表 |
| `status` | string | 用户状态（ACTIVE/INACTIVE） |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

## JWT 令牌结构

### 令牌格式

后端使用自定义 JWT 实现（HS256 算法），令牌格式：

```
header.payload.signature
```

### JWT Payload 示例

```json
{
  "sub": "admin",              # 用户标识（user_id）
  "iat": 1704067200,           # 签发时间戳
  "exp": 1704153600,           # 过期时间戳
  "iss": "workflow-api",       # 签发者
  "aud": "workflow-client"     # 接收者
}
```

### JWT 配置

后端 JWT 配置（在 `app/shared/db/config.py`）：

```python
JWT_SECRET_KEY: str = "your-secret-key"           # JWT 密钥
JWT_EXPIRE_MINUTES: int = 60 * 24                # 令牌有效期（分钟）
JWT_ISSUER: str = "workflow-api"                 # 签发者
JWT_AUDIENCE: str = "workflow-client"            # 接收者
```

## 错误响应

### 常见错误码

| HTTP 状态码 | 错误信息 | 说明 |
|-------------|----------|------|
| 400 | validation error | 请求参数校验失败 |
| 401 | user not found | 用户不存在 |
| 401 | invalid credentials | 用户名或密码错误 |
| 401 | token expired | 令牌已过期（后续 API 调用） |
| 403 | permission denied | 权限不足（后续 API 调用） |

### 错误响应示例

```json
{
  "code": 401,
  "message": "invalid credentials",
  "data": null
}
```

## 后续 API 调用

### 使用 JWT 令牌

登录成功后，在后续 API 请求的 Header 中携带令牌：

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 示例代码（前端）

```typescript
// 设置认证头
const authHeaders = {
  'Authorization': `Bearer ${accessToken}`,
  'Content-Type': 'application/json'
};

// 调用需要认证的 API
const fetchRequirements = async () => {
  const response = await fetch('/api/v1/requirements', {
    headers: authHeaders
  });
  // 处理响应...
};
```

## 密码处理

### 后端密码存储

- 密码使用 bcrypt 或自定义哈希算法存储
- 数据库中存储 `password_hash` 和 `password_salt` 两个字段
- **绝对不能**存储明文密码

### 前端密码传输

- 密码通过 HTTPS 传输（生产环境必须启用 TLS）
- 不在客户端本地存储密码明文
- 可选择性地在 `localStorage` 中存储 `rememberMe` 状态

## 完整登录流程

### 1. 前端登录

```typescript
// 1. 用户输入用户名和密码
const loginForm = {
  username: 'admin',
  password: '123456',
  rememberMe: false
};

// 2. 转换为后端格式并发送请求
const response = await fetch('/api/v1/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    user_id: loginForm.username,
    password: loginForm.password
  })
});
```

### 2. 后端验证

```python
# 后端服务层验证逻辑
async def authenticate_user(user_id: str, password: str) -> Dict[str, Any]:
    # 1. 查找用户
    user = await UserDoc.find_one(UserDoc.user_id == user_id)

    # 2. 检查用户状态
    if not user or user.status != "ACTIVE":
        raise UserNotFoundError("user not found")

    # 3. 验证密码
    if not verify_password(password, user.password_salt, user.password_hash):
        raise ValueError("invalid credentials")

    # 4. 返回用户信息
    return self._doc_to_dict(user)
```

### 3. 返回令牌

```python
# 后端路由处理
@router.post("/login", response_model=APIResponse[LoginResponse])
async def login(request: LoginRequest, service: RbacServiceDep):
    # 1. 验证用户
    user = await service.authenticate_user(request.user_id, request.password)

    # 2. 生成 JWT
    token = create_access_token(user["user_id"])

    # 3. 返回令牌和用户信息
    return APIResponse(
        data=LoginResponse(
            access_token=token,
            user=UserResponse(**user)
        )
    )
```

### 4. 前端处理响应

```typescript
// 4. 前端处理登录响应
const result = await response.json();

if (result.code === 200) {
  // 登录成功
  const { access_token, user } = result.data;

  // 存储令牌（根据 rememberMe 决定存储位置）
  if (loginForm.rememberMe) {
    localStorage.setItem('access_token', access_token);
    localStorage.setItem('user_info', JSON.stringify(user));
  } else {
    sessionStorage.setItem('access_token', access_token);
    sessionStorage.setItem('user_info', JSON.stringify(user));
  }

  // 更新应用状态
  setCurrentUser(user);
  setAccessToken(access_token);
  setView('req_list');  // 跳转到主页
} else {
  // 登录失败，显示错误信息
  setLoginError(result.message);
}
```

## 注意事项

### 安全建议

1. **HTTPS 传输**: 生产环境必须使用 HTTPS 加密传输
2. **密码策略**: 建议密码最小长度 6 位，支持复杂密码策略
3. **令牌过期**: JWT 设置合理过期时间（如 24 小时）
4. **令牌刷新**: 考虑实现令牌刷新机制
5. **记住我**: `rememberMe` 仅影响前端本地存储策略，不影响后端令牌有效期
6. **错误信息**: 不要在错误信息中暴露敏感信息（如"用户不存在"vs"密码错误"）

### 最佳实践

1. **输入验证**: 前端和后端都要进行输入校验
2. **错误处理**: 统一错误处理和错误信息格式
3. **日志记录**: 记录登录成功/失败日志
4. **并发控制**: 防止暴力破解（可考虑添加验证码或请求限制）
5. **敏感信息**: 不在客户端日志中输出密码或令牌
