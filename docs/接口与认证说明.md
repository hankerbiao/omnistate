# 接口与认证说明

更新时间：2026-03-17

## 文档范围

本文档合并原“后端接口说明”和“认证与登录指南”，作为根目录下唯一的接口接入说明。

## 1. 通用约定

- API 前缀：`/api/v1`
- 数据格式：`application/json`
- 认证方式：JWT Bearer Token

统一响应 Envelope：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

字段说明：

- `code`：成功为 `0`，失败为 HTTP 状态码或业务码
- `message`：成功默认 `ok`
- `data`：成功时为业务数据，失败时为错误详情或 `null`

错误响应 `data` 示例：

```json
{
  "error": "ValidationError",
  "detail": "title is required"
}
```

## 2. 健康检查

### `GET /`

- 用途：基础健康检查
- 返回：`{ "status": "ok", "message": "Workflow API 服务运行中 (MongoDB)" }`

### `GET /health`

- 用途：健康检查
- 返回：`{ "status": "healthy", "message": "Service is running" }`

### `GET /health/ready`

- 用途：就绪检查

### `GET /health/live`

- 用途：存活检查

## 3. 登录与当前用户

### 登录 `POST /api/v1/auth/login`

请求体：

```json
{
  "user_id": "admin",
  "password": "123456"
}
```

成功响应 `data`：

```json
{
  "access_token": "<jwt-token>",
  "token_type": "Bearer",
  "user": {
    "user_id": "u-1003",
    "username": "admin",
    "email": "admin@example.com",
    "role_ids": ["ROLE_ADMIN"],
    "status": "ACTIVE",
    "created_at": "2026-02-28T00:00:00Z"
  }
}
```

### 当前用户 `GET /api/v1/auth/me`

- 请求头：`Authorization: Bearer <access_token>`
- 用途：获取当前登录用户信息

## 4. 前端集成约定

令牌存储建议：

- `rememberMe=true`：写入 `localStorage`
- `rememberMe=false`：写入 `sessionStorage`

前端关键接入点：

- `frontend/src/services/api/TestDesignerApi.ts`
- `frontend/src/services/api/index.ts`
- `frontend/src/services/api/ApiClient.ts`
- `frontend/src/App.tsx`
- `frontend/src/components/views/Login.tsx`

要求：

- 统一由 API Client 注入 `Authorization` 头
- 应用初始化时恢复令牌与用户态
- 未配置后端地址时，允许显式回退到本地模拟登录

## 5. 常见错误

### `401 Unauthorized`

- 后端未启动
- `VITE_BACKEND_API_BASE_URL` 配置错误
- 用户名或密码错误
- JWT 配置不一致

### 请求未携带令牌

- 未执行 `setAccessToken(...)`
- API Client 未注入 `Authorization`

### 刷新后丢失登录状态

- `rememberMe` 分支未正确写入存储
- 应用启动时未恢复令牌和用户信息
