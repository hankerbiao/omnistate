# 测试需求字段快速参考

## 字段一览表

| 序号 | 字段名 | 字段类型 | 必填 | 示例值 | 说明 |
|------|--------|----------|------|--------|------|
| 1 | **req_id** | string | ✅ | `"TR-2026-001"` | 需求唯一ID，格式：`TR-YYYY-NNN` |
| 2 | **title** | string | ✅ | `"DDR5内存测试"` | 需求标题，简洁明确 |
| 3 | **description** | string | ✅ | `"详细描述..."` | 需求详细描述 |
| 4 | **technical_spec** | string? | ❌ | `"JEDEC DDR5标准..."` | 技术规格（可选） |
| 5 | **target_components** | string[] | ✅ | `["Memory","CPU"]` | 目标组件列表 |
| 6 | **firmware_version** | string? | ❌ | `"BIOS v2.1.0"` | 固件版本（可选） |
| 7 | **priority** | enum | ✅ | `"P0"` | 优先级：P0/P1/P2 |
| 8 | **key_parameters** | {key,value}[] | ✅ | `[{"key":"电压","value":"1.1V"}]` | 关键参数 |
| 9 | **risk_points** | string? | ❌ | `"高温下ECC错误率..."` | 风险点（可选） |
| 10 | **tpm_owner_id** | string | ✅ | `"tpm001"` | TPM负责人ID |
| 11 | **manual_dev_id** | string | ✅ | `"tester001"` | 手工执行负责人ID |
| 12 | **status** | enum | ✅ | `"DEVELOPING"` | 状态：PENDING/DEVELOPING/REVIEWING/CLOSED |
| 13 | **attachments** | Attachment[] | ✅ | `[{"name":"spec.pdf"}]` | 附件列表 |
| 14 | **created_at** | string | ✅ | `"2026-01-15T08:00:00Z"` | 创建时间 |
| 15 | **updated_at** | string | ✅ | `"2026-02-20T10:30:00Z"` | 更新时间 |

---

## 枚举值详解

### Priority (优先级)
- **P0**: 最高优先级 - 关键路径测试，影响上线
- **P1**: 高优先级 - 重要功能测试
- **P2**: 普通优先级 - 常规测试

### RequirementStatus (需求状态)
- **PENDING**: 待指派 - 刚创建，等待指派TPM和执行人
- **DEVELOPING**: 用例开发中 - TPM和执行人已指派，正在编写用例
- **REVIEWING**: 评审中 - 用例已提交，等待评审
- **CLOSED**: 已闭环 - 评审通过，需求完成

## 目标组件枚举

可选择的目标组件：
- **Memory** - 内存
- **CPU** - 处理器
- **Storage** - 存储设备
- **Network** - 网络设备
- **GPU** - 显卡
- **BMC** - 主板管理控制器
- **BIOS** - 固件
- **Thermal Sensor** - 温度传感器
- **Power Supply** - 电源
- **Motherboard** - 主板

---

## 状态流转

```
PENDING → DEVELOPING → REVIEWING → CLOSED
    ↓           ↓           ↓
指派负责人   提交评审     评审通过
```

## 表单验证要点

### 1. 字符长度
- title: 1-100字符
- description: 1-2000字符
- technical_spec: ≤2000字符
- risk_points: ≤1000字符

### 2. 格式要求
- req_id: 建议 `TR-YYYY-NNN`
- 时间: ISO 8601 格式
- firmware_version: `组件名 版本号`

### 3. 业务规则
- target_components: 不允许重复
- key_parameters: key和value都不能为空
- tpm_owner_id ≠ manual_dev_id
- 所有ID必须真实存在
