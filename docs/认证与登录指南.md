## 接口与数据结构

### 登录接口

- 方法：`POST /api/v1/auth/login`
- 请求体：

```json
{
  "user_id": "admin",
  "password": "123456"
}
```

- 成功响应（Envelope 内 data）：

```json
{
  "access_token": "<jwt-token>",
  "token_type": "Bearer",
  "user": {
    "user_id": "u-1003",
    "username": "admin",
    "email": "admin@example.com",
    "role_ids": ["ROLE_ADMIN"],
    "status": "ACTIVE",
    "created_at": "2026-02-28T00:00:00Z"
  }
}
```

### 获取当前用户

- 方法：`GET /api/v1/auth/me`
- 认证：`Authorization: Bearer <access_token>`

### 关键字段说明

- 登录表单字段：`user_id`、`password`、`rememberMe`
- 后端认证字段：`user_id`、`password`
- `rememberMe=true`：令牌写入 `localStorage`
- `rememberMe=false`：令牌写入 `sessionStorage`

## 环境配置

在 `frontend/.env` 中配置：

```bash
VITE_BACKEND_API_BASE_URL=http://localhost:8000
VITE_API_TIMEOUT_MS=15000
```

## 快速验证流程

### 1. 启动后端

```bash
cd backend
python -m app.main
```

健康检查：

```bash
curl http://localhost:8000/health
```

### 2. 启动前端

```bash
cd frontend
npm install
npm run dev
```

### 3. 验证登录

1. 访问前端页面并输入账号密码登录。
2. 打开浏览器开发者工具，确认登录请求命中 `/api/v1/auth/login`。
3. 检查后续业务请求是否包含 `Authorization: Bearer <token>`。

## 常见问题

### 登录 401

- 检查后端是否正常启动。
- 检查前端 `VITE_BACKEND_API_BASE_URL` 是否正确。
- 检查账号是否存在、密码是否正确。
- 检查后端 JWT 相关配置是否一致。

### 请求未带令牌

- 检查是否执行了 `setAccessToken(...)`。
- 检查 `ApiClient` 是否通过 `getAuthToken` 注入令牌。

### 刷新后丢失登录状态

- 检查 `rememberMe` 分支是否正确写入存储。
- 检查应用初始化是否读取并恢复令牌与用户信息。
