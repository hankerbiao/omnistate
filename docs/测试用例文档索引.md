## 📊 测试用例字段分类

### 按功能分类

#### 1. 基本信息（4 个字段）
- case_id, ref_req_id, title, test_category

#### 2. 版本与状态（4 个字段）
- version, is_active, change_log, status

#### 3. 责任人（3 个字段）
- owner_id, reviewer_id, auto_dev_id

#### 4. 优先级与执行（2 个字段）
- priority, estimated_duration_sec

#### 5. 测试目标与环境（2 个字段）
- target_components, required_env

#### 6. 标签与条件（3 个字段）
- tags, pre_condition, post_condition

#### 7. 测试步骤（2 个字段）
- steps, cleanup_steps

#### 8. 自动化相关（4 个字段）
- is_need_auto, is_destructive, automation_type, script_entity_id

#### 9. 风险与可见性（3 个字段）
- risk_level, visibility_scope, confidentiality

#### 10. 附件与扩展（2 个字段）
- attachments, custom_fields

#### 11. 故障与废弃（2 个字段）
- failure_analysis, deprecation_reason

## 🔄 需求与用例关联

### 关系说明
- **需求**（TestRequirement）是**测试用例**（TestCase）的父级对象
- 一个需求可以包含多个测试用例
- 用例通过 `ref_req_id` 字段引用需求的 `req_id`

### 对比示例

| 字段 | 需求 | 用例 |
|------|------|------|
| ID | req_id | case_id |
| 标题 | title | title |
| 描述 | description | change_log（变更日志） |
| 目标组件 | target_components | target_components |
| 状态 | RequirementStatus（4个值） | TestCaseStatus（4个值） |
| 优先级 | Priority（P0/P1/P2） | Priority（P0/P1/P2） |
| 负责人 | tpm_owner_id, manual_dev_id | owner_id, reviewer_id, auto_dev_id |

## 🚀 使用建议

### 📝 开发者
1. **首次使用**: 从 [`用例字段快速参考.md`](./用例字段快速参考.md) 开始
2. **深入理解**: 阅读 [`测试用例字段定义.md`](./测试用例字段定义.md)
3. **实现编码**: 参考 API 端点和验证规则

### 🧪 测试人员
1. **创建用例**: 使用快速参考表的最小必填字段清单
2. **编写步骤**: 参考 TestStep 结构示例
3. **提交评审**: 遵循状态流转规则

### 🎯 TPM/项目经理
1. **理解流程**: 阅读工作流状态说明
2. **审批用例**: 使用审批历史记录功能
3. **管理需求**: 结合测试需求文档一起使用

### 🔍 查找信息
1. **快速查找**: 使用字段一览表
2. **详细说明**: 使用详细定义文档
3. **关联查询**: 参考需求与用例关联部分
