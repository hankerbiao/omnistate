# 后端接口说明

## 统一响应格式
所有接口统一返回标准 Envelope：
```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

字段说明：
- `code`：业务码，成功为 `0`，失败为 HTTP 状态码。
- `message`：成功默认 `ok`，失败为错误语义说明。
- `data`：业务数据，失败时为错误详情对象或 `null`。

错误响应 `data` 结构：
```json
{
  "error": "ErrorName",
  "detail": "human readable message"
}
```

## 健康检查

### `GET /`
- 说明：基础健康检查。
- 响应 `data`：`{ "status": "ok", "message": "Workflow API 服务运行中 (MongoDB)" }`

### `GET /health`
- 说明：健康检查。
- 响应 `data`：`{ "status": "healthy", "message": "Service is running" }`

### `GET /health/ready`
- 说明：就绪检查。
- 响应 `data`：`{ "status": "ready", "message": "Service is ready to accept requests" }`

### `GET /health/live`
- 说明：存活检查。
- 响应 `data`：`{ "status": "alive", "message": "Service is alive" }`
