## 3. req_id 约束收敛（需求创建）

### 3.1 目标

创建需求时，`req_id` 必须由后端生成，前端不得提交。

### 3.2 最终约束链路

1. 前端类型约束：`CreateRequirementPayload` 明确排除 `req_id`。
2. 前端提交约束：创建请求不构造 `req_id` 字段。
3. 路由入口检查：若检测到 `req_id`，记录警告并继续忽略。
4. 服务层强制覆盖：无论是否传入，都重新生成 `req_id`。
5. 数据库唯一索引：`req_id` 物理唯一。

### 3.3 生成规则

- 格式：`TR-YYYY-XXX`（例如 `TR-2026-001`）
- 按年递增编号
- 保留并发下的唯一性防线（服务层生成 + DB 唯一索引）

### 3.4 结论

`req_id` 的控制权已下沉到后端与数据库层，前端仅消费生成结果。
