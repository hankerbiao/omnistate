# 测试执行集成方案

更新时间：2026-03-17

## 文档范围

本文档合并原“测试任务下发与进度回传设计方案”和“执行进度回传 SDK 设计方案”，统一描述执行域、回调协议与 SDK 形态。

## 1. 目标

- 从平台下发测试任务到外部框架
- 接收任务、用例、步骤级进度回传
- 以最小改造成本提供 SDK 接入
- 保证回调幂等、可追踪、可审计

## 2. 执行域边界

- `test_cases`：定义“测什么、怎么测”
- `execution`：管理“下发结果、执行进度、执行结果”
- `workflow`：继续负责业务事项流转，不直接承载执行过程细粒度状态

建议模块目录：

```text
backend/app/modules/execution/
  api/
  schemas/
  service/
  repository/
```

## 3. 核心对象

### ExecutionTask

主任务表，记录：

- `task_id`
- `external_task_id`
- `framework`
- `dispatch_status`
- `overall_status`
- `request_payload`
- `dispatch_response`
- `dispatch_error`
- `case_count`
- `reported_case_count`
- `started_at`
- `finished_at`
- `last_callback_at`

### ExecutionTaskCase

任务下用例明细，记录：

- `task_id`
- `case_id`
- `case_snapshot`
- `status`
- `progress_percent`
- `step_total`
- `step_passed`
- `step_failed`
- `step_skipped`
- `last_seq`
- `last_event_id`

### ExecutionEvent

回调审计事件，记录：

- `task_id`
- `event_id`
- `event_type`
- `seq`
- `received_at`
- `source_time`
- `raw_payload`
- `processed`
- `process_error`

## 4. 状态约定

### 下发状态

- `PENDING`
- `DISPATCHED`
- `DISPATCH_FAILED`

### 任务总体状态

- `QUEUED`
- `RUNNING`
- `PASSED`
- `FAILED`
- `PARTIAL_FAILED`
- `CANCELLED`
- `TIMEOUT`

### 用例状态

- `QUEUED`
- `RUNNING`
- `PASSED`
- `FAILED`
- `SKIPPED`
- `BLOCKED`
- `ERROR`

乱序保护：

- 仅处理 `seq > last_seq` 的事件
- 重复或过期事件只留审计，不覆盖当前状态

## 5. 平台 API

统一前缀：`/api/v1/execution`

### 任务下发

- `POST /tasks/dispatch`

请求体应包含：

- `framework`
- `callback_url`
- `dut`
- `cases`
- `runtime_config`

返回核心字段：

- `task_id`
- `external_task_id`
- `dispatch_status`
- `overall_status`
- `case_count`

### 查询接口

- `GET /tasks/{task_id}`
- `GET /tasks/{task_id}/cases`
- `GET /tasks`

### 回调接口

- `POST /callbacks/progress`

推荐请求头：

- `X-Framework-Id`
- `X-Event-Id`
- `X-Timestamp`
- `X-Signature`

支持事件类型：

- `TASK_STATUS`
- `CASE_STATUS`
- `STEP_RESULT`
- `HEARTBEAT`
- `SUMMARY`

## 6. SDK 设计

首期建议：

- Python SDK：`dml-exec-reporter`
- 语义化版本：`MAJOR.MINOR.PATCH`
- 内部 PyPI 或私有源发布

SDK 负责：

- 生成标准 payload
- 计算 HMAC 签名
- 调用回调接口
- 自动生成 `event_id`
- 重试、退避、可选落盘缓存

SDK 不负责：

- 判定测试通过与否
- 任务编排调度
- 平台侧聚合规则

## 7. Python SDK 最小接口

```python
from dataclasses import dataclass

@dataclass
class ReporterConfig:
    base_url: str
    framework_id: str
    secret: str
    timeout_sec: float = 3.0
    max_retries: int = 5


class ExecutionReporter:
    def report_task_status(self, task_id: str, status: str, seq: int, **kwargs) -> None: ...
    def report_case_status(self, task_id: str, case_id: str, status: str, seq: int, **kwargs) -> None: ...
    def report_step_result(self, task_id: str, case_id: str, step_id: str, status: str, seq: int, **kwargs) -> None: ...
    def heartbeat(self, task_id: str, seq: int, **kwargs) -> None: ...
    def summary(self, task_id: str, overall_status: str, seq: int, totals: dict, **kwargs) -> None: ...
    def flush(self, timeout_sec: float | None = None) -> None: ...
    def close(self) -> None: ...
```

## 8. 落地顺序

1. 新增 `execution` 模块及主表/明细表/事件表
2. 落地 `POST /tasks/dispatch` 与查询接口
3. 落地 `POST /callbacks/progress`，先保证幂等与审计
4. 提供 Python SDK 参考实现
5. 前端接任务看板和任务详情
